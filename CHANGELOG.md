# Changelog

Все значительные изменения этого проекта будут задокументированы в этом файле.

## [0.1.0] - 2026-06-01

### Добавлено
- **Основной API сервер на FastAPI** - инициализирована базовая структура приложения
- **Эндпоинт корневой страницы** (`GET /`) - возвращает приветственное сообщение
- **Эндпоинт получения всех студентов** (`GET /students`) - получение списка всех студентов с возможностью фильтрации по курсу
  - Поддерживает опциональный параметр `course` для фильтрации по номеру курса
- **Эндпоинт получения студентов по курсу** (`GET /students/{course}`) - детальная фильтрация студентов с множественными параметрами
  - Параметр `course` - номер курса (обязательный)
  - Параметр `major` - специальность (опциональный)
  - Параметр `enrollment_year` - год поступления
- **Утилиты для работы с JSON** (`utils.py`)
  - Функция `json_to_dict_list()` - преобразование JSON файлов в список словарей
  - Функция `dict_list_to_json()` - преобразование списка словарей в JSON и сохранение в файл
- **Тестовые данные** - файл `students.json` с примерами данных студентов
- **Требования к зависимостям** - файл `req.txt` с необходимыми пакетами

### Примечания
- Начальная версия с базовой функциональностью для работы со студентами
- Данные хранятся в JSON файле
- API полностью функционален и готов к использованию

## [0.2.0] - 2026-06-01

### Добавлено

* **Pydantic-модель студента** (`Student`) - добавлена схема для описания и проверки данных студента

  * Проверка типов данных
  * Проверка корректности email через `EmailStr`
  * Ограничение длины строковых полей
  * Проверка курса студента в диапазоне от 1 до 5
  * Проверка года поступления
  * Проверка номера телефона
  * Проверка даты рождения

* **Перечисление специальностей** (`Major`) - добавлен ограниченный список доступных специальностей

  * `Информатика`
  * `Экономика`
  * `Право`
  * `Медицина`
  * `Инженерия`
  * `Языки`

* **Модель обновления студента** (`StudentUpdate`) - добавлена отдельная схема для изменения курса и специальности студента

* **Эндпоинт добавления студента** (`POST /students`)

  * Принимает данные нового студента в формате JSON
  * Проверяет данные через Pydantic-модель
  * Проверяет уникальность `student_id`
  * Добавляет нового студента в `students.json`

* **Эндпоинт обновления студента** (`PUT /students/{student_id}`)

  * Находит студента по `student_id`
  * Обновляет курс и специальность студента
  * Сохраняет изменения в `students.json`
  * Возвращает ошибку `404`, если студент не найден

* **Эндпоинт удаления студента** (`DELETE /students/{student_id}`)

  * Находит студента по `student_id`
  * Удаляет студента из списка
  * Сохраняет изменения в `students.json`
  * Возвращает ошибку `404`, если студент не найден

### Изменено

* Логика работы с данными стала более строгой за счет валидации через Pydantic
* Функция записи данных в JSON теперь используется для сохранения изменений после добавления, обновления и удаления студентов

### Примечания

* Завершена вторая итерация проекта
* API теперь поддерживает базовые CRUD-операции для работы со студентами
* Данные по-прежнему хранятся в JSON-файле
* База данных, авторизация и полноценные тесты пока не реализованы

## [0.3.0] - 2026-06-01

### Добавлено
- Подключена инфраструктура PostgreSQL через Docker Compose.
- Добавлен сервис PostgreSQL для локальной разработки.
- Добавлен Adminer для просмотра таблиц базы данных через веб-интерфейс.
- Добавлен файл `app/config.py` для чтения настроек подключения из `.env`.
- Добавлен файл `app/database.py` с настройкой SQLAlchemy engine, session maker и базового класса `Base`.
- Добавлены SQLAlchemy-модели `Student` и `Major`.
- Настроен Alembic для управления миграциями базы данных.
- Создана первая миграция для таблиц `students` и `majors`.
- Проверена связь между таблицами через внешний ключ `major_id`.

### Изменено
- Проект подготовлен к переходу от хранения данных в `students.json` к PostgreSQL.

### Примечания
- API-эндпоинты пока продолжают работать через `students.json`.
- Работа эндпоинтов напрямую с PostgreSQL будет реализована в следующей итерации.

## [0.4.0] - 2026-06-01

### Добавлено
- **DAO-слой** для работы с базой данных
  - Добавлен базовый класс `BaseDAO`
  - Добавлены `StudentDAO` и `MajorDAO`
  - Реализованы методы получения, добавления, обновления и удаления записей через SQLAlchemy

- **Роутер студентов и специальностей**
  - Добавлен файл `app/students/router.py`
  - Эндпоинты вынесены из `app/main.py` в отдельный роутер
  - Подключение роутера выполнено через `app.include_router()`

- **Pydantic-схемы для PostgreSQL-версии API**
  - Добавлены схемы `MajorCreate`, `MajorRead`
  - Добавлены схемы `StudentCreate`, `StudentRead`, `StudentUpdate`
  - Добавлена поддержка преобразования SQLAlchemy-моделей в API-ответы через `ConfigDict(from_attributes=True)`

- **Эндпоинты для работы со специальностями**
  - `GET /majors`
  - `POST /majors`

- **Эндпоинты для работы со студентами через PostgreSQL**
  - `GET /students`
  - `POST /students`
  - `PUT /students/{student_id}`
  - `DELETE /students/{student_id}`

### Изменено
- `app/main.py` очищен от старой логики работы с JSON
- Основные API-эндпоинты переведены с `students.json` на PostgreSQL
- Временный префикс `/db` удалён, маршруты стали основными
- Работа с данными теперь проходит через связку `router -> DAO -> PostgreSQL`

### Удалено
- Удалена старая JSON-логика работы со студентами
- Удалён файл `students.json`
- Удалён файл `utils.py`
- Удалён старый файл `app/schemas.py`, который использовался для JSON-версии API

### Примечания
- Завершён переход от хранения данных в JSON-файле к PostgreSQL
- API теперь использует базу данных как основной источник данных
- Авторизация и полноценные тесты пока не реализованы
